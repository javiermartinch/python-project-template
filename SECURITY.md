# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest release | :white_check_mark: |
| Previous release | :x: |

## Reporting a Vulnerability

**Do NOT open a public issue for security vulnerabilities.**

Please report security vulnerabilities privately via GitHub Security Advisories:

1. **GitHub Security Advisories:** Use [Report a vulnerability](../../security/advisories/new) on this repository.

If you maintain a repository generated from this template, publish a real monitored
private reporting channel before making that repository public.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial Assessment:** Within 5 business days
- **Fix & Disclosure:** Coordinated disclosure within 90 days

We follow [responsible disclosure](https://en.wikipedia.org/wiki/Responsible_disclosure) practices.
